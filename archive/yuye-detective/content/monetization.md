## 4. Day 7 後的付費轉換文案

### 4.1 設計原則：解鎖「檔案」的感覺，不是逼課金
- 出現時機：**僅在結局演出完整播完、分享圖生成之後**，以「偵探社檔案櫃」的畫面呈現，不打斷情緒。
- 視覺語言：上鎖的牛皮紙檔案袋，標籤寫著檔案名——是「補完」的隱喻，不是「你被擋住了」的隱喻。
- 玩家關閉後不再彈出；付費入口常駐於結局後選單，安靜存在。

### 4.2 轉換文案（normal_end 後，繁中）

> 案件結束了。結衣平安，千夏被捕。
> 但偵探社的檔案櫃裡，還留著一份你沒有打開的檔案——
> 關於那七封信真正的主人，關於三年前那個雨夜，關於他/她為什麼說「我不是第一次失去你」。
>
> 【True Case File】這個謊的結局，可以由你來寫。
>
> （主線已全部免費完整呈現。此檔案是給想知道「全部」的你的補完，不影響你已獲得的結局。）

### 4.3 轉換文案（bad_end 後，繁中）

> 偵探社的燈熄了。
> 也許哪裡走錯了——也許重走一次雨夜，你會看見不一樣的線索。
>
> 【免費重玩一次】或【打開 True Case File，看看你錯過了什麼】

（bad_end 後**優先推免費重玩**，付費入口放第二位。失敗當下推銷是情緒勒索的高風險區。）

### 4.4 禁用話術清單與替代寫法

| 禁用話術 | 為什麼禁 | 替代寫法 |
|---|---|---|
| 「付費才是真結局」 | 貶低免費結局＝變相閹割承諾 | 「normal_end 是完整的結局。True Case File 是同一夜的更深處。」 |
| 「不付費我就消失」「偵探在等你救他」 | 以角色情感勒索付費 | 「故事已經完整說完了。如果你還想多陪他/她一個雨夜，檔案在這裡。」 |
| 「限時 24 小時，錯過不再」 | FOMO 壓迫 | 不設限時。可寫「檔案不會消失，你隨時可以回來。」 |
| 「只剩最後 N 份」 | 數位商品假稀缺＝欺騙 | 一律不用數量話術 |
| 「全部玩家都買了」 | 從眾壓力＋通常是假數據 | 不引用購買數據 |
| 「你忍心讓結衣／偵探……嗎？」 | 把付費包裝成道德責任 | 付費描述只談「你會看到什麼」，不談「角色會怎樣」 |
| 以彈窗反覆打斷 | 騷擾式轉換 | 每結局後最多出現一次，之後僅常駐選單 |

## 5. Stripe Payment Links 對接（無後端版）

### 5.1 流程
1. Stripe Dashboard 建立 4 商品 × 4 幣別的 **Payment Links**（測試模式設計，**不建真實商品**）。
2. 每條 Link 設定付款成功後導向自架的靜態成功頁，例如：
   `https://<site>/pay/success-true-case.html`
3. 成功頁（純靜態）顯示對應商品的**解鎖碼**（例：`TCF-RAIN-0617`），並附操作說明。
4. 玩家回到 `unlock.html` 輸入解鎖碼 → 前端 JS 驗證（雜湊比對，不放明文碼於 JS）→ 寫入 `localStorage`（如 `unlocked: ["true_case_end"]`）→ 對應結局/檔案頁解鎖。

```
玩家 → Stripe Payment Link → 付款成功 → 靜態 success 頁（顯示解鎖碼）
     → 回 unlock.html 輸入 → JS 雜湊驗證 → localStorage → 內容解鎖
```

技術備註：story.json 中付費結局文本若直接放明文，看原始碼即可讀到。可接受的輕量防護：付費結局文本以解鎖碼衍生金鑰做簡單對稱加密（XOR/AES-lite）存放，輸入正確碼才能解密渲染。這擋不住有心人，但擋得住「F12 直接看」。

### 5.2 風險：解鎖碼可被分享

- **事實**：成功頁是靜態的、碼是固定的（每商品一碼，非一次性）。任何買過的人都能把碼貼到網路上。
- **為什麼可接受**：
  1. 本作客單價 NT$60–250，盜用造成的損失上限極低；做防盜的工程成本（後端、帳號系統）遠高於損失。
  2. 主線本來就免費，付費者買的是「補完＋支持創作」的心情，這群人分享碼的動機低。
  3. 碼外流的傳播效果接近「免費試讀」，對 Season 2 反而是曝光。
  4. 零後端＝零個資、零伺服器成本、零被入侵面，這個交換划算。
- **低成本緩解**：
  - 每商品每「季」換一次碼（改 unlock.html 的雜湊＋success 頁顯示文字即可）。
  - 解鎖碼格式帶語義（`TCF-RAIN-0617`），方便日後追蹤外流批次。
  - unlock.html 加輸入頻率限制（前端 debounce＋錯 5 次冷卻 60 秒），防暴力枚舉。

## 6. Optional：Cloudflare Worker + KV 版本（webhook 驗證、發碼）

需要「一次性解鎖碼」時的升級路徑。玩家端仍是靜態頁，只多兩個 fetch。

### 6.1 架構

```
Stripe Payment Link
   │ checkout.session.completed (webhook)
   ▼
Cloudflare Worker /webhook
   1. 驗 Stripe-Signature（webhook secret）
   2. 產生一次性碼 code = random(12)
   3. KV.put("code:"+code, {product, used:false, created})
   4.（可選）以 Stripe 收據 email 寄碼；或導向帶 session_id 的領碼頁
   ▼
玩家在 unlock.html 輸入碼
   ▼
Worker /redeem
   1. KV.get 查碼 → 驗 product、未用過
   2. KV 標記 used:true（綁定首次兌換的隨機裝置指紋 token）
   3. 回傳該商品的解密金鑰（付費文本仍存於前端，加密）
   ▼
前端解密渲染 → localStorage 記錄已解鎖
```

### 6.2 元件職責
- **Worker `/webhook`**：唯一信任入口。必須驗簽（`stripe.webhooks.constructEvent` 等價的 WebCrypto HMAC 驗證），拒絕無簽名請求。
- **Worker `/redeem`**：唯一發鑰匙的 API。Rate limit（同 IP 每分鐘 ≤ 10 次）。失敗回應一律同語句（不洩漏「碼存在但已用」vs「碼不存在」）。
- **KV**：只存 `code → {product, used, created, device_token}`。**不存任何玩家個資**（email 交由 Stripe 持有，不落地）。
- **領碼頁**（可選）：`/claim?session_id=...` → Worker 向 Stripe API 驗 session 已付款 → 顯示碼。避免依賴 email。

### 6.3 取捨
| | 無後端版（§5） | Worker+KV 版 |
|---|---|---|
| 碼可分享 | 是（接受） | 否（一次性） |
| 月成本 | $0 | $0（免費額度內）|
| 個資風險 | 零 | 近零（不存 email） |
| 失敗模式 | 無 | Worker 掛掉＝買了拿不到碼（需客服補發流程） |

建議：Season 1 用無後端版上線；若單月銷量 > 500 筆或碼外流明顯影響營收，再升級 Worker 版。

## 7. 防止 server 成本爆掉的規則

1. **靜態優先（鐵律）**：玩家端 100% 靜態 HTML/CSS/JS＋story.json＋localStorage。託管用 Cloudflare Pages / GitHub Pages，頻寬流量成本 $0，爆紅也不會爆帳單。
2. **玩家端零 AI、零 API key**：所有對話內容預先生成寫進 story.json。沒有任何執行期 LLM 呼叫＝不存在「每個玩家都在燒 token」的成本模型，也不存在 key 外洩風險。
3. **Worker 免費額度防線**（若採 §6）：
   - Cloudflare Workers 免費層 10 萬請求/日、KV 免費層 10 萬讀/日 1 千寫/日——webhook＋redeem 的真實流量遠低於此（寫入次數 ≈ 銷量）。
   - **不綁信用卡於 Cloudflare 帳號**（或設 $0 付費上限），超額時 Worker 直接回 429 而非產生帳單。
   - Worker 上設告警：日請求 > 1 萬（正常量的異常倍數）時通知，視為被打。
   - `/redeem` 強制 rate limit；webhook 端點路徑不公開、僅接受帶有效簽名的請求。
4. **圖片/分享圖**：分享圖在前端 canvas 生成（零伺服器），靜態素材壓縮後經 CDN 快取。
5. **禁止項**：不接任何「按玩家數計費」的第三方服務（分析用 Cloudflare Web Analytics 免費版即可）；不做即時排行榜、雲端存檔等需要資料庫的功能。

## 8. 退款與客服基本文案（繁中）

### 8.1 退款政策（商品頁與成功頁皆連結）

> **退款政策**
> - 付款後 **14 天內**，若解鎖碼無法使用、或內容與商品說明不符，請來信申請全額退款。
> - 已正常解鎖並閱讀內容的情況，因數位商品特性，原則上不提供退款；但若你認為內容與說明有落差，仍歡迎來信，我們會個案處理。
> - 退款一律退回原付款方式（透過 Stripe 處理，約 5–10 個工作天入帳）。
> - 申請時請附上：付款收據編號（Stripe 收據 email 內）＋你購買的商品名稱。**請勿提供信用卡號**，我們不需要也不會索取。

### 8.2 客服信箱自動回覆

> 你好，這裡是雨夜偵探社的信箱（非自動偵探，是真人看信）。
> 我們已收到你的來信，會在 **3 個工作天內**回覆。
> 常見問題：
> 1. **解鎖碼輸入沒反應？** 請確認大小寫與「-」符號完整，並關閉瀏覽器的無痕模式（無痕模式關閉後解鎖紀錄會消失）。
> 2. **換裝置/清除瀏覽器資料後解鎖消失？** 解鎖紀錄存在你的瀏覽器中。請保留付款收據，來信附上收據編號即可協助重新解鎖。
> 3. **退款？** 請見退款政策，14 天內符合條件全額退。
> 雨夜漫長，感謝你的耐心。
