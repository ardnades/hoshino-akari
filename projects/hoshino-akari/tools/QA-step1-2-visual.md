# Day4 + Day6 演出層人工目視 QA Checklist

> 範圍：Step 1（camera 原語＋未讀紅點）＋ Step 2（Day6 回收）。**只目視，不改碼。**
> 狀態目標：把 `visual-pending` 推到 `visual-pass`。
> 緣由：headless 環境無顯示器，camera/紅點的視覺體感只能人工實機確認。

## 0. 怎麼跑

1. 本機開 `projects/hoshino-akari/play/index.html?dev`（`?dev` 才有開發者面板）。
2. 按 `d` → 開發者面板 → 點 **Day 4**（紅點/camera 主戰場）；Day6 同法跳 **Day 6**。
3. 每個環境各跑一次（見下表）。記錄 PASS/FAIL。

## 1. 環境矩陣（每天各跑一次）

| 代號 | 環境 | 怎麼設 |
|---|---|---|
| P | 手機直幅 | 真手機或 DevTools 裝置模擬（如 390×844）|
| D | 桌機寬幅 | 一般視窗最大化 |
| S | 快進 skip | 遊戲內「» 快進」開 |
| R | reduced-motion | OS 開「減少動態」/ DevTools Rendering → prefers-reduced-motion: reduce |

## 4. 九項跨場 gate（最關鍵）

| 代號 | 檢查 | FAIL 樣態 |
|---|---|---|
| G1 破圖 | push 1.04/1.08、pan＋scale 在 P 手機直幅下背景不露白邊、立繪臉不被切難看 | 邊緣黑/白條、臉貼框 |
| G2 紅點搶戲 | 1 顆=注意、2+=壓力，但不像錯誤警告、不像「必須點開的任務」 | 紅點比角色表情先吸眼 |
| G3 hold 殘留（最高風險） | 每個 scene/day/ending 後 framing 回正；Day5 開頭無殘留推近或偏移 | 下一場仍是放大/偏移 |
| G4 pause 節奏 | 0.8s/2.0s 是「壓力」不是「當機」；S skip 下不被長 pause 卡住 | 玩家以為遊戲停住 |
| G5 UI 不被推 | 對白框不縮放、選項不偏移、SNS 卡不被推走、紅點固定右上不隨 zoom、點擊區不位移 | 任一 UI 跟著 stage 變形 |
| G6 R/S 收斂 | R 下 camera duration≈0；S 下無中途 transform 殘留、跳場後已 reset、紅點 pulse 不延遲爆出 | 殘留/延遲 |
| G7 資訊過載 | 同屏 SNS＋震動＋camera＋pause＋紅點時，重點次序＝角色情緒 > 手機壓力 > SNS | 一瞬間太多動畫看不出重點 |
| G8 指令掛對行 | 紅點/camera 出現在「對的那句」，不是文字與動畫同時太快冒出 | 錯位/搶拍 |
| G9 旗標存活 | （程式已確認）Day4 觸發未讀 → Day6 仍讀得到回收段 | Day6 回收段沒出現 |
