# 41 — 立繪缺口清單（演出驅動的新立繪需求）

> 來源：2026-06-19 一次跨全劇研究 pass（每日一支 agent 逐行掃 Day1~7＋chapters＋endings，共標出 62 個 beat，合成為 23 張提案）。
> 目的：把「劇情演出需要、但現有立繪做不到」的缺口，整理成可排程、可走產線的新立繪清單。
> 配套：做法見 `40-立繪演出與角度立繪製作做法.md`；canon 鎖定見記憶 `akari-character-lock-v3`、`akari-fullbody-master`、`akari-sprite-staging-and-angles`。

## 1. 現況與三大結構缺口

現有立繪全是「大腿以上正面 bust」＋ 3 張背/側離場圖。整類動作／角度／全身都缺，導致作者一遇做不出的姿勢就只能 `expr:""` 把立繪藏掉（玩家看到「人憑空消失」），或停在打架的表情上。

三大結構性缺口：

1. **完全沒有全身遠景** — 入場、離席、踢石子、吃完丟垃圾全靠旁白帶過，人物憑空出現／消失（Day2/4/5/7 至少 5 處）。
2. **沒有動作姿勢** — 蹲、坐、捧物、吃東西全做不到，只能藏立繪或停在不搭的笑臉。
3. **角度太薄** — 只有「離場純背影」，缺「回頭帶表情」「並肩同向」「3/4 側戒備望門」（Day6 雨棚並肩是最長的立繪真空段）。

兩個明確錯接（非缺、是修）：

- Day7 第一次素顏正視（`妝容＋塌髮＋貓眼神`）被接到 `akari_cb_shy.png`（別開視線）＝方向相反。
- 整個 hidden 結局＋三結局訪談剪輯缺「演出服無口罩」立繪，現只能 `expr:""` 藏人。

## 7. 接線與分工慣例

- **幾何**：所有新立繪固定 832×1216 去背 PNG；`@masks` overlay 必須原生同幾何對位。
- **正面 bust**＝疊 `@masks` 口罩/半罩 overlay；**角度立繪（背/側/蹲/並肩）**＝口罩烤入底圖、不疊 frontal overlay。
- **引擎接點**：以 `line.expr` 字串為鍵（單一 sprite 槽）；新姿勢多半「一張通用底＋引擎 expr 切換」即可覆蓋多 beat，差分非必要。搭配既有演出欄位 `pos/depth/motion(walk_in/step_in…)/clear`。
- **與 loop CG 的分工**：`ev_reveal`(D1:13 揭臉) / `ev_signlight`(D2:119 招牌光) / `ev_umbrella`(D6:135 雨夜並肩) / `ev_lipbalm`(D7:110 護唇膏對視) 四支 `.mp4` 已覆蓋各高潮幀；以上立繪服務的是這些 CG **前後的對話/移動** beat，不重複。
