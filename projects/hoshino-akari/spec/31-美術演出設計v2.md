# 星野灯線《偷來的七天》美術演出設計 v2

> **定位**：本檔承接 v1《[30-美術素材需求清單](30-美術素材需求清單.md)》的素材盤點，升級為**可實裝、可演出、可販售**的商業 AVG 美術演出規格（接近《アマガミ》《Muv-Luv》「用立繪演戲」的路數）。內容＝1 份 KEY REGISTRY 索引 ＋ 6 份交付。
>
> **演出總綱**：立繪扛約 95% 的日常戲（鬥嘴／嘴硬／別開視線／靠近退後／並肩走路／經理人說明行程一律走立繪＋overlay＋背景 pan＋SNS UI＋item popup＋空鏡＋BGM 降音＋text_speed）；**CG 只用於高潮／記憶點／宣傳／回收**。主角不做立繪、不露臉，只在 CG 以第一人稱出現手／前臂／肩／背身輪廓（最多到脖子以下）。
>
> **狀態標記**：✅ 已接素材（assets.js 已綁或 `_AKARI` 已產 baked 圖）／🟡 SVG 佔位待製（art.js 已有 fallback、cg key 已被觸發，換正式圖零改碼）／⬜ 全新待製（無圖、多需新增 key 或契約變更）。
>
> **製作工法**：**工法 1** ＝ exprlock 緊臉差分（既有 master 低 denoise 0.40–0.60 masked img2img，outside-mask diff≈0、color_match＋iris_relock，保證同一張臉）；**工法 2** ＝ 全新構圖（無 master，Anima-native IP-Adapter＋ControlNet-LLLite＋固定 seed＋identity/iris QC）。全程走 Anima 3-piece loader：底模 `anima_baseV10` 為 Cosmos-Predict2-2B DiT，**禁用 SDXL／Flux／Qwen-Image 的完整流程**；但 loader 仍借用 `qwen_3_06b_base` 文字編碼器與 `qwen_image_vae`（Anima 既定組態，不等於走 Qwen-Image 管線）。
>
> **校驗原則（全檔共通）**：「劇本既有 expr」**僅指實際出現在 `play/data/*.js` 的字串**；本檔與 `spec/*.md` 內出現的 expr 命名一律視為**待新增**，落地需改 story data（非僅 assets）。
>
> **canon 紅線（違反即廢稿）**：經理人非反派；星野灯任何狀態同一臉骨；怒＝可愛悶氣（嘴硬／別開視線／小抿嘴／可微紅，禁惡狠狠瞪）；Day6 同框風險非戀愛散步；Day7 對視非告白；不用無敘事 CG 稀釋主高潮。
>
> **⚠ 商用授權風險（最高層級 open decision）**：本案美術管線模型 **Anima Base v1.0 為非商用授權**，與「可販售」目標衝突。僅非商業發布（demo／作品集）暫不觸發；轉為付費販售即觸發——**量產散布前必須先關閉此 gate**（詳見交付 6）。
>
> **本 v2 審查結論**：通過 canon 審查（**0 違規**）＋ Anima 管線審查（**0 錯誤**），validator **據前次回報為 0 error / 0 warning（落地前必須重新實跑 `node projects/hoshino-akari/tools/validate_story.js` 並附 stdout）**；已套用 5 項準確性修正（經理人 expr 契約分級、`face_smile_signlight` 改 cg 行號 day2.js:118–119、`eye_contact_rooftop` 新獨立 key 勿覆蓋 day7:111、`back_eating_oden` 實裝錨點 day1.js:91、spec 命名校驗原則）。本檔僅為設計規格，**未改動任何 `.js`**；所有「契約變更」項落地前須授權＋跑 `node projects/hoshino-akari/tools/validate_story.js`。

## ⑤ 經理人 who/expr 接法

**機制事實（已確認）**：`M.names.manager`（meta.js:41）= `{label:"經理人", cls:"manager"}` **已存在** → `who:"manager"` 今天就是 validator 合法值（story_schema.js:134 `line-who` 不噴警告）。劇本若已有 `who:"manager"` 行，引擎名牌正常顯示。`setSprite("manager", expr)` 會查 `characters.manager[expr]` — 但 `assets.js` 的 `manager:{}` 仍**註解掉** → 目前必走 sprite-hidden fallback（名牌有字、無立繪，不穿幫）。

> **關鍵校驗（重要，先讀再接）**：經理人立繪**並非「填 assets 即 drop-in」**。原因是 `day7.js` 經理人 S2 那幾行（L47–62 區）目前**沒有 `expr:` 字串**——`setSprite("manager", expr)` 收到的 `expr` 是 `undefined`／空字串，`characters.manager[expr]` 永不命中，光填 `assets.js` 不會出圖。`expr_mgr_neutral`（＝spec/registry 命名）**只存在於 spec markdown，`play/data/*.js` 並無此字串** → 依全檔共通校驗原則，它是「待新增」，落地必須先改 story data。因此經理人 S2 立繪是**契約變更（story data）**，需兩步、缺一不可：
>
> 1. **改 `day7.js` 經理人 S2 行（L47–62 擇要）**：補 `expr:"平穩職業不冷不兇"`（或改採其他既有命名；勿直接寫 registry 的 `expr_mgr_neutral` 英文 key，engine 不認）。
> 2. **改 `assets.js`**：在 `characters.manager` 對應**該中文字串**填圖路徑。
>
> 兩步缺一則永不出圖。**真正「填 assets 即 drop-in」的只有 `主持人`／`工作人員`**——這兩個 expr 字串 `endings.js` 已實際出現（已有 `expr:`），故補 `assets.characters.manager["主持人"]`／`["工作人員"]` 即生效，無須改劇本。
