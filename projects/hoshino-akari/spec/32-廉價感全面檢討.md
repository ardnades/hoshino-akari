# 32 —— 廉價感全面檢討（Cheapness Audit）

> 日期：2026-06-16 JST。範圍：player runtime 全體（不只 Day4/Day6 切片）。
> 觸發：演出層 Step 1/2 完成後，整體仍有廉價感。
> 方法：對照實際 `play/` 程式與素材（非空談）。
> 一句話判定：**廉價感已從「演出層缺席」轉移到「美術 register（視覺語言）不統一」。** 程式／演出／資料三層已達 A−，卡關在美術資產層與少數呈現細節。

---

## 0. 先更正一個常見誤判

- **音訊不是廉價來源。** `data/assets.js` 的 `enabled:{bgm:true,se:true}`，且 `assets/se/` 有 36 個 mp3、`assets/bgm/` 4 軌都在。演出時是真的有聲。
- 唯一音訊弱點：BGM 僅 4 軌（night/rain/store/warm）要撐 7 天多場景，長時間會聽出循環。屬 P2 微弱項，不是主因。

## 1. 根因清單（依「對廉價感的貢獻度」排序）

每條格式：**現況 → 檔案佐證 → 為何廉價 → 嚴重度 / 所屬層**。

### C1. 道具／結局／圖鑑 CG 全是 SVG 簡筆線稿　★★★★★（美術資產）
- 現況：所有 CG（油豆腐/布丁/熱可可/護唇膏/收據/紙片/攝影棚/四結局徽記）都是 inline SVG 單色描邊塗鴉。
- 佐證：`play/art.js` 檔頭自述「全部用 inline SVG 線稿…歪歪的手繪貓 + 暖色細線」；`assets.js` 的 `cg:{}` 為空 → 引擎一律走 SVG fallback。
- 為何廉價：背景是寫實 webp、立繪是 Anima 算出的真人半身，CG 卻是線稿小圖。**寫實 × 塗鴉並置＝立刻露餡**，讀作 placeholder。而診斷把「四記憶錨點 CG＋畫廊」列為頭號商品賣點，賣點現在長得像 debug 佔位圖。
- 嚴重度：最高。這是「道具圖難看」的根因。

### C2. 字幕把主題寫死在圖上　★★★★☆（美術資產＋設計）
- 現況：每張 CG 的 SVG 內嵌 `<text>`：「最油的那塊・自己准的」「焦糖布丁・普通很貴」「圖不用字・比較像回信」。引擎另在 `showCG()` 又疊一層 `.cg-cap`（`capFor(key)`）。
- 佐證：`art.js` 各 CG 的 `<text>`；`engine.js showCG()` + `capFor()`。
- 為何廉價：(a) 劇本台詞已表達同義 → 冗餘；(b) 把主題當字幕貼圖＝同人習作手法，是 show-don't-tell 的反面；(c) 同一句可能出現兩次（baked + cg-cap）。
- 嚴重度：高，且部分可程式立刻改（見 Q1）。

### C3. 「表情變化」一大半是假的　★★★★☆（美術資產）
- 現況：多個表情名指向同一張立繪。`口罩圍巾／壓低帽簷／拉低帽簷／喘氣帽簷低／警戒／戒備遮臉／圍巾遮臉／看玻璃門眼神警戒` 全部 → `maskon_v001.png`；素顏群多對一。
- 佐證：`data/assets.js` characters 對照表。
- 為何廉價：夜戲（戴口罩）整段名牌上的「表情」字一直換，但立繪是同一張臉一動不動。玩家會察覺「動的是文字、圖沒動」。靜止立繪是 VN 廉價感經典來源。真正有差異的只有 `cb_` 那組。
- 嚴重度：高。

### C4. 真正的高潮畫面「不存在」　★★★★☆（美術資產）
- 現況：診斷點名的宣傳主視覺——D1 揭臉三秒、D6 並肩半步、D7 對視、四結局——一張真 CG 都沒有，全 fallback。
- 佐證：`assets.js cg:{}` 空；`art.js` 僅徽記。
- 為何廉價：最該截圖瘋傳的瞬間，畫面沒有可截的圖。「賣點靠截圖」的 VN 沒有圖可截。
- 嚴重度：高（賣點層）。

### C5. 標題畫面與圖鑑是塗鴉＋emoji　★★★☆☆（美術資產＋程式）
- 現況：標題主圖 `ART.title()` 是 SVG 貓塗鴉＝玩家第一眼；回想室是 SVG 道具格 + 鎖頭，其中 `locked: () => '🔒'` 直接用 emoji。未讀紅點 icon 我也用了字元「✉」。
- 佐證：`art.js` title/locked；`engine.js` unreadBadge 的 `✉`。
- 為何廉價：第一印象就是塗鴉；emoji 鎖頭與 ✉ 在一個講究的 VN 裡讀作偷懶。
- 嚴重度：中（但「第一印象」權重高）。

### C6. 立繪只有一個基本姿態、無肢體語言／無待機微動　★★★☆☆（美術資產）
- 現況：核心立繪是單一「neutral guarded」半身＋口罩 overlay＋少數 cb 裁切。無姿勢變化、無待機呼吸。
- 為何廉價：站樁式立繪。即使換表情也只是換貼圖、不會「呼吸」。
- 嚴重度：中。部分可程式緩解（見 Q4 待機微動）。

### C7. 演出幅度過於客氣，核心仍「讀字→點擊」　★★★☆☆（程式／設計）
- 現況：camera push 1.04～1.08 在手機上幾乎看不出；紅點是唯一明確新可見元素。
- 為何廉價：Step1/2 補上了動力學，但幅度小到容易被讀成「沒事發生」。骨子裡仍是打字機＋點擊。
- 嚴重度：中。可程式調整，但**注意陷阱**（見第 4 節）：不能靠「加更多演出」硬蓋，否則變「漂亮的塗鴉播放器」。

### C8. 選項「重要 vs 語氣」對玩家仍不可見　★★☆☆☆（程式，已排程 Step 3）
- 現況：Step 2 已標 `choiceType:"flavor"`（d3door/d5s4/d7s7），但 UI 上一視同仁。
- 為何廉價：玩家無法分辨哪些選項有後果，容易覺得「選什麼都一樣」＝假互動感。
- 嚴重度：中低，已有 Step 3 計畫。

### C9. 名牌旁印「表情字」、轉場份量輕　★★☆☆☆（程式／設計）
- 現況：`engine.js` 在角色名牌旁用 `.sp-expr` 印出 `node.expr`（如「微笑」）；行間轉場僅 4px swap fade；場景僅小 toast。
- 為何廉價：把情緒用文字標在名牌＝與立繪冗餘、且像 debug 標籤；轉場無重量。
- 嚴重度：低。可程式調整。

### C10. camera 把道具 CG 一起推歪（Step1 bug）　★★☆☆☆（程式 bug）
- 現況：`#cgLayer` 被放進 camera transform 群組，push/hold 狀態下顯示 CG 會被放大偏移（Day4 護唇膏、Day6 油豆腐）。
- 佐證：`style.css` `#spriteLayer,#cgLayer{transform…}`；Day4/Day6 接線殘留 push。
- 嚴重度：低（但讓 C1 雪上加霜）。屬既知 review finding，應修。

## 3. 可程式立刻改善的 Quick Wins（不需美術、additive、可驗證）

> 這些都能在不產任何新圖的前提下，降低廉價感或止血。每項都應跑 validate + smoke 維持綠燈。

- **Q1（C2）遊戲內 CG 隱藏字幕。** `showCG()` 不再疊 `.cg-cap`（或加開關），並評估移除 `art.js` 各 CG 的 `<text>`。圖鑑要不要保留說明文字另議。效果：道具一出現不再自帶字幕，廉價感立降。
- **Q2（C10）修 camera 推歪 CG。** 把 `#cgLayer` 移出 camera transform 群組（道具/結局圖永遠穩定置中）。並補 Day4 告別前、Day6 暖場前各一個 `camera:reset`，消除 push 殘留。
- **Q3（C5）去 emoji。** `locked` 的 `🔒` 改成既有 `lock()` SVG；未讀 `✉` 改成簡單 SVG 信封或純圓點，與整體描邊風格一致。
- **Q4（C6）立繪待機微動。** 對靜止立繪加極輕的「呼吸」位移／縮放（CSS，幅度 < 1%，reduced-motion 關閉）。讓「圖在動」，緩解站樁感。注意與 camera/ depth 不衝突。
- **Q5（C9）名牌表情字降噪。** 移除或弱化名牌旁 `.sp-expr` 的情緒字（避免「8 個名字同一張臉」被字面點破），或只在素顏場顯示。
- **Q6（C7）camera 幅度小幅上修＋給「份量」轉場。** push small/medium 微調、關鍵拍給稍長 duration；行間/場景轉場加一點點重量（但見陷阱）。
- **Q7（C8 / Step 3）flavor 選項淡樣式。** engine 讀 `choiceType` 加 class + CSS 淡樣式（淡但不像 disabled、不顯字、不加 icon）。

## 4. 陷阱・不該做（重要）

1. **不要再靠「加更多 camera／紅點／演出」蓋廉價感。** 根因是美術 register，不是演出量。再加演出只會讓「漂亮的塗鴉播放器」更忙、更亂（違反診斷與 G7 資訊過載）。
2. **不要為了統一風格把寫實背景/立繪降級成塗鴉。** 方向是把 CG 升級到寫實，不是把全體降到線稿。
3. **camera 幅度校準要排在美術升級之後。** 先把 CG/立繪做對，再調鏡頭幅度，否則會對著佔位圖瞎調。
4. **嚴守既有契約**（CLAUDE.md no-touch）：上述 Quick Wins 不得改 judge/route/ending/save 格式；CG 字幕、emoji、待機微動皆屬呈現層，additive。

## 5. 美術管線待產清單（Anima/ComfyUI，非程式）

> 引擎 `showCG` 已「真圖優先、缺圖 fallback」，路徑 `assets/cg/<key>.png` 現成，產完即用、不需改 runtime。

1. 四記憶錨點 CG：`oden / pudding / cocoa / lipbalm`（無字幕、與立繪同畫風、寫實暖色）。
2. 高潮 CG：D1 揭臉、D6 並肩半步、D7 對視；四結局收尾圖。
3. 暗號 CG：`receipt`（收據雙面）、`note`（標籤縫紙片）——本作 USP「圖不用字」，最該做精。
4. 表情立繪變體：至少口罩場的眼神/眉差異 2–3 款，打破「多對一同一張臉」。
5. 標題主視覺；四結局徽記改設計過的 icon。
6. （選）BGM 增軌。

## 6. 驗收方向

- 美術項：CG 出現時與背景/立繪同一視覺 register，無「佔位感」；關鍵高潮有可截圖的成品。
- 程式 Quick Wins：validate 0/0、四結局 smoke 可達；CG 不再被 camera 推歪、不再自帶字幕；無 emoji；待機微動在 reduced-motion 下關閉。
- 全體手感：玩家不再覺得「動的只有文字／圖是佔位」。

## 7. 建議下一步

1. 先做 **P0 程式止血**（Q1 去遊戲內字幕、Q2 修 CG 推歪＋補 reset、Q3 去 emoji）——不需美術、立即降廉價感。
2. 平行啟動 **P0 美術**：四記憶錨點真 CG。
3. 之後才 P1（表情變體、高潮 CG）→ P2（演出幅度校準、Step 3 flavor 樣式、待機微動）。

## 8. 第二輪追加根因 C11–C24（成品感／佈局／文字）

> 第一輪 C1–C10 是「畫面」廉價（美術 register）。本輪挖的是「功能成品感／佈局／文字」廉價——玩家未必說得出，但會直覺「這沒做完」。

### 成品感基礎建設
- **C11　字型未內嵌 web font　★★★（程式）**：`:root --font-serif` 靠系統字型堆疊。Mac/iOS 有 Noto Serif TC/PingFang 好看；Android/Windows 無明體時退到黑體或泛用 serif → 跨裝置長相不一致。無 `@font-face` 內嵌＝第一印象交給運氣。VN 質感八成在字。
- **C13　無設定畫面　★★★（程式）**：選單只有 回到遊戲/回想室/重玩本日/回標題。無音量（BGM 0.5、SE 0.6 寫死）、無文字速度、無 auto 速度。標準 VN config 缺席。
- **C14　無歷史/回顧 backlog　★★★（程式）**：已加「全畫面點擊前進」，但誤點即跳行且無法回看。對白是 A 級資產卻無重讀機制，既傷體驗又顯未完成。
- **C19　head 裸奔　★★★（程式，易修）**：head 只有 stylesheet。無 favicon（分頁預設地球圖示）、無 `theme-color`（手機頂欄不上色）、無 manifest/apple-touch-icon、**無 `og:`（分享連結一片空白）**——對想靠截圖/分享傳播的 VN 是硬傷。近乎零成本可補。
- **C21　無資源預載、無載入狀態　★★（程式）**：12 個同步 `<script>`，首場景背景/立繪無 preload → 慢網時文字先出、背景晚一拍 pop-in，空窗廉價；亦無 loading 指示。
- **C16　BGM 硬切、無淡入淡出　★★（程式）**：`playBGM` 換 src 直接 play、音量寫死、`bgm:"stop"` 立即暫停，mood 切換音樂硬切。
- **C15　回標題用 `location.reload()`　★★（程式）**：整頁白閃重載而非乾淨狀態重置。
- **C17　存檔單槽、以「天」為單位　★★（程式/設計）**：無中途存檔/多槽/縮圖，無法保留多路線——對「四結局＋重玩」賣點偏薄。
- **C13b 無障礙　★（程式）**：`user-scalable=no` 關閉縮放（併入 C24）。

### 佈局
- **C18　無橫向(landscape)佈局　★★★（程式）**：`@media` 只處理寬窄/高度，無 `orientation` 分支。手機橫拿時可視高度變矮、textbox 仍吃 30–38vh、立繪 `height:92%` 變得又矮又窄 → 擠成一條。
- **C22　選項全螢幕模糊罩　★★（程式）**：`#choices`(z8) backdrop blur 把整個場景糊掉，2 選項也整片遮，偏重。
- **C23　無「隱藏文字框看圖」　★（程式）**：VN 標配缺席，想純看背景/CG 沒辦法。
- **C24　`user-scalable=no` 關閉縮放　★（程式）**：無障礙性偏弱。

### 文字／結局
- **C20　母題與句式過度重複　★★★（劇情/設計）**：實證 Day2「我又摸到它。今天第幾次了。」＋章節 intro/outro 每天繞「護唇膏／數日子／又一次」。護唇膏事實鏈是對的，但「又…」句式與「還剩 N 天」獨白七天反覆，讀來同質。演出層能分隔視覺，句式重複仍在。
- **C12（修正）　結局差異化偏弱，非份量薄　★★★（劇情/資料）**：四結局各約 26 行（warm 8–35、quiet 36–62、bitter 63–89、hidden tail 90–104）。份量其實尚可；真正問題是 **bitter 結構≈quiet、hidden＝warm＋14 行尾巴**，差異化不足，玩家重玩四結局時「分歧感」不夠。

## 9. 更正（誠實修訂）

- **桌機固定框存在**：`#app{max-width:920px;margin:0 auto}` 已把桌機限成 920px 置中欄，**故「桌機像放大手機頁」不成立，撤回**。
- **響應式非全無**：有 `@media`（寬窄/高度）、`safe-area-inset`、`prefers-reduced-motion`、`@media(hover:none)`。缺口是 orientation（C18）與字型內嵌（C11）。
- **C12 由「太短」修正為「差異化弱」**（見上）。

## 10. 三主軸總表（可派工）

廉價感收斂成三條主軸：

| 主軸 | 涵蓋 | 需美術？ | 可程式獨立完成？ | 槓桿 |
|---|---|---|---|---|
| **A. 美術 register** | C1 CG塗鴉・C2 字幕・C3 表情多對一・C4 高潮無圖・C5 標題/圖鑑/emoji・C6 站樁立繪・C10 CG被推歪 | 多數**需美術** | C2/C10/部分 C5 可程式止血 | 最高（賣點畫面） |
| **B. 成品感基礎建設** | C11 字型・C13 設定・C14 回顧・C15 回標題・C16 BGM淡入・C17 存檔・C19 head・C21 預載 | **不需美術** | **全部可程式** | 高 CP（試做↔成品分界） |
| **C. 文字與結局差異化** | C12 結局差異・C20 母題句式重複 | 不需美術 | **你能直接補（劇情/資料）** | 高（落在玩家下總評的時刻） |

### 派工建議
