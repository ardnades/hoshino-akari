# 星野灯線《偷來的七天》廉價感全面檢討 v1

> **定位**：本檔不是「劇本診斷」（那是 [20-診斷與強化建議](20-診斷與強化建議.md)），也不是「美術需求」（[30](30-美術素材需求清單.md)/[31](31-美術演出設計v2.md)）。本檔檢討的是**已實裝、玩家實機跑得到的成品**，為什麼仍會被人一眼看成「免費文字播放器／同人練習作」，以及每一條的**已研究解法**（守住純靜態網頁、玩家端零後端、輕量 runtime、繁中、商用授權 gate）。
>
> **方法**：7 個維度各派一個 sub-agent 讀實際源碼稽核、再各派一個研究員查商業/網頁 VN（Ren'Py / TyranoScript / Monogatari）做法產出解法，最後跨維度綜合。所有 file:line 證據與「死 flag、重複檔、授權矛盾」皆已對 `play/` 源碼親驗（grep / md5 / ffprobe 級）。
>
> **誠實前提（最重要）**：這個專案的 CSS／演出／美術底子**遠高於免費播放器下限**——純 CSS mood 分層、立繪演出 v0（pos/depth/motion/walk_in/rush_in/shake＋競態守衛＋reduced-motion）、SNS 覆蓋層、未讀紅點、五區回想室、19 張同 pipeline 水彩背景、12 張同臉骨去背立繪、SE 淡出、缺素材一律 fallback 不壞。**問題不是「沒做」，是「幾個高權重位置沒收尾」與「設計意圖在多數玩家機器上看不到」。** 本檔第六節專列「已部分處理、勿當缺陷灌水」。
>
> **本檔不落地任何 `.js`**：純檢討＋解法規格。任一解法落地前須依 CLAUDE.md 走對應流程（No-Touch 需授權、改 data 跑 `node projects/hoshino-akari/tools/validate_story.js`、改 JS 跑 `node --check`、改文字跑 `scripts/check-text.ps1`）。

## 第一節 · 一句話結論

> 文字 A 級、CSS／演出／美術底子是「半成品的商業級」；廉價感**不來自單點 bug，而來自 4 條根因的投影**——(A) 字型沒打包，承載 90% 體驗的文字在多數玩家機器上退成系統黑體；(B) 線稿貓 placeholder 與真水彩並置在標題/結局/收集牆等最高權重位置，自證未完工；(C) 演出與音訊引擎缺「過場交疊」與「混音」兩個抽象層，所有轉場都是硬切；(D)「選了沒差＋漏看找不回＋不能調」三缺口疊加，宣告這是 demo 不是商品。**修根因比逐條修更省工。**

## 第二節 · 系統性根因（4＋1 條）

全部 30+ 條發現是這幾條根因的投影。逐條修是 30 個工單；修根因是 5 個。

### 根因 A — 沒打包字型（一次缺口，五個維度各記一次）
**貫穿：字型排版／視覺／VN 完成度。** `index.html:7` 整個 `<head>` 只有一行 stylesheet，全專案 **0 個** `@font-face` / Google Fonts / preconnect（已親驗）。`style.css:19` 的 `--font-serif` 首選 `Noto Serif TC`/`Source Han Serif TC` 等**預設不隨 Windows/Android 出貨**的明體名，`body` 套此堆疊（`style.css:25`）。實機結果：絕大多數玩家（Win Chrome/Edge、Android Chrome）退到 serif 關鍵字 → 新細明體/被映成黑體；Apple 明體只有 Mac/iPhone 開發者本人看得到。
這一件事同時引爆 `font-no-webfont`、`font-sans-same-trap`、`VIS-5`、`ART-7`、`vn-webfont-missing` 五條，並讓 `.02em` 偏緊字距、無避頭尾、無 smoothing 全在「沒被設計過的退字」上放大。**一次修復（嵌一套 OFL 明體 woff2 subset + preload）可同時關閉 5–8 條，全表 ROI 最高。**

### 根因 B — 兩種美術完成度在同一畫面並置，自證「未完工」
**貫穿：視覺／美術。** 真水彩 CG/立繪（商業級）與 `art.js` 手繪線稿貓 SVG（placeholder 級）**並排在玩家權重最高的四個位置**：標題（`engine.js:565`→`ART.title()`，340px 大）、四結局徽記（`engine.js:474` `ART[em.badge]()`）、hidden 結局唯一獨佔 CG（`akari_studio` 仍是 SVG，`assets.js` cg{} 無此 key）、回想室 gallery（`engine.js:492` 真 CG 與線稿貓同格網格）。涉及 `VIS-1/4`、`ART-1/5`。**回想室把真水彩和線稿貓擺在一起，等於遊戲自己證明「那隻貓是佔位圖」。** 修法是現有水彩 pipeline 的延伸，非新技術。

### 根因 C — 引擎缺「過場」與「混音」兩個抽象層，所有轉場硬切
**貫穿：演出引擎／音訊。** BGM 換軌（`engine.js:25`）是裸 `src=url; play()` 無 ramp；`mood:"stop"`（`engine.js:23`）直接 `pause()` 硬停。背景 `setMood` 對單一 `#moodbg` 同步重指 `backgroundImage`，**`showScene` 註解寫「讓背景 crossfade 起頭」但程式只有 `delay(220)`，沒有任何交疊圖層**（`PERF-02`，註解與實作不符特別危險）。無 ducking、音量全程寫死（BGM 0.5 / SE 0.6）。
諷刺點（已親驗）：**SE 做了 ~100ms 淡出（`engine.js:34-38`），更該淡的 BGM 反而沒有。** 缺一個雙層交疊背景＋一個 BGM 音量 ramp 函式（兩個小抽象），就讓後巷→店內→雨棚→攝影棚每次轉場像 PPT 翻頁＋音樂啪一聲。這是「高級小說，不是 AVG」體感的技術總源頭。

### 根因 D —「選了沒差＋漏看找不回＋不能調」三缺口疊加
**貫穿：VN 完成度／互動／音訊。**
- **選了沒差**：`d7_food`（`day7.js:169/178`）、`tag_banter`/`tag_honest`（`day6.js:113/124`）各只 set、**全 repo 0 讀取**（已親驗 grep；`M.judge` 不讀這三個；`endings.js` 四後日談完全不讀 flag）。最傷的是 `d7_food` 在倒數第二個選擇、緊接結局，動機強的玩家最會回頭驗。
- **漏看找不回**：無 backlog/history（`engine.js` 無 history 結構、`index.html` 無入口）。
- **不能調**：無設定畫面，音量/文字速度/自動速度全寫死；快進是**無差別全略過**（非「快進到未讀」，無已讀追蹤）。
- **守不住**：驗證器僅 89 行，自承「可達性分析未做（TODO）」（`validate_story.js:73`），0 errors PASS 卻放行上述死 flag。
三者互相放大：誤觸快進飛掉未讀劇情 → 沒 backlog 可救 → 也沒設定可預防。而驗證器缺 set↔read 對稱檢查，是假選項能溜進成品的**製造源**。

### 根因 E（次級）— 版面與資產未收尾
**貫穿：視覺／美術。** `#app{max-width:920px}`＋body 純黑 `#0b0e13`，RWD **無任何 >920px 桌機規則**（`VIS-2`）→ 寬螢幕上是一條手機直幅漂在死黑側欄；閉眼笑/閉眼立繪 1052×1494 vs 系統 832×1216，`object-fit:contain` 下換表情臉位會跳（`VIS-3`/`ART-3`）；`oden.png` 與 `oden_cup.png` 位元組相同（已親驗 MD5 `d0437ee…`）。同屬「資產與版面未經整理」。

## 第三節之二 · 優先序修訂（整合 GPT 第二意見，2026-06-18）

> 諮詢網頁版 ChatGPT 後的修訂。GPT **未讀 repo**（自評「確實性中高」），故其架構洞見採用、事實判斷以本檔對源碼的親驗為準。原第三節 Top 10 保留作對照，**實作以本節為準**。

### 採用 GPT 的修正（比原版更好）
- **placeholder 升 P0**：最會被截圖的位置（標題/結局徽記/hidden CG/gallery）放 placeholder，比十個小 UI bug 更傷——這正是根因 B，原本卻排第 5。升 P0-1。
- **placeholder 用 reuse 而非生新圖**：以現有 19 張水彩裁切＋排版＋徽記設計做「完成品感替代圖」，**先不為補圖而狂生新 Anima CG**。修正原 #5「用 Anima pipeline 生新圖」會踩非商用 gate 的問題——reuse 路線 license-safe。
- **音量併入 Config，勿單列高位**：單一 slider 只是「沒那麼失禮」，非質感核心。
- **不做完整 ducking/mixer，先 fade**：與本檔原把 ducking 列次優先一致。
- **授權＝商業發售 P0 gate**：採用 `assets_manifest.json`（asset id／path／source／model／license／commercial usable／credit required／replacement needed）。
- **死 flag validator 規則**：`flag written but never read = error`、`choice without immediate feedback = warning`、`late-game choice without ending/afterword effect = error`。落地即把 `story_schema.js:265-328 analyzeRelations()`（已寫好、未接 CLI，見 INT-3）接上。

### 修正 GPT 的事實誤判（這些其實已存在，勿重做）
- **Title 選單／Continue／Gallery／Auto／Skip／遊戲內選單已實裝**（`engine.js` boot/選單）。GPT 以為全缺。真正缺口＝多槽 Save/Load、Config、Backlog、Credits。外殼約 6 成在。
- **GPT 的 nodeId/lineIndex 中途續玩 schema**：現引擎需重構 `runDay/playNodes` 才能做（＝ vn-single-save 階段二，高成本 No-Touch）。非「localStorage 就夠」；MVP 先做 day 粒度多槽。
- **「文字節奏演出語法」非新漏項**：是 spec/20 全域主題＋PERF-04。
- **UI「沒有狀態」不實**：hover/active/picked/disabled/觸控分支已有（`style.css:273-286`）；只缺 `focus-visible`（a11y）。
- **Gallery 已有 caption＋locked 斜紋剪影**（`ART.lock()`），非空格 debug 牆；解鎖註記是加分非補洞。
- **部分手機細節已處理**：`user-select:none`、`maximum-scale=1`、`safe-area-inset`（`style.css:23,27`、`index.html:5`）；缺的是 `100dvh`/iOS 底欄/橫放。

- **Loading/preload/first-frame 管理**：loading overlay、font/首屏 BG/首立繪 preload、title 首擊解鎖 audio、loaded 才進 title（消白屏/字跳/BG 黑一下/BGM 第一聲播不出）。

## 第四節 · 逐維度發現與解法

> 嚴重度：**H**＝玩家一眼可辨廉價/業餘；**M**＝老玩家會扣分；**L**＝細節。「半套」＝已部分處理。解法均守純靜態/無後端/輕量；牽涉授權者於第七節彙整。

### 4.1 中文字型與排版
**已做好**：serif/sans/num 三套分工正確、內文 `clamp()` 響應字級、`line-height:1.85`、reduced-motion 已顧。問題在「整套設計依賴玩家機器很可能沒有的字型，且零 web font 兜底」。

| id | 嚴重 | 證據 | 為何廉價 | 解法摘要 |
