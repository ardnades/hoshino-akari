## 已修

### [x] 1. 出 CG 時有一瞬間黑畫面
- **現象**：切到 event CG 時，畫面會閃一下全黑再出圖。
- **根因**：CG 圖未載完就先把 cgLayer 亮出來（暗底瞬間蓋上 → 空窗黑幀）。
- **修法（engine.js `showCG` + style.css `#cgLayer`，2026-06-22）**：
  1. `#cgLayer` 整層（含暗底）一起 `cgIn` 淡入，不再讓暗底瞬間蓋上（前一 commit）。
  2. `showCG` 改 async：**靜圖先 `await img.decode()` 解碼完才上畫面**，杜絕「清黑底→才載圖」的空窗；加 `cgToken` 防呆（解碼途中玩家已換頁/clear → 放棄舊圖，不蓋到新畫面）。
- **驗證**：preview 實跑——prop CG `oden_solo.png` 出現當下 `complete=true / naturalWidth=1254`（已解碼才現身），無黑幀；無 console error。
- **限制**：影片 CG（loop `.mp4/.webm`，如 ev_reveal / cat_meet）無法 `img.decode()`，仍走 `poster` 靜圖墊底；poster 缺檔時退回現狀。要徹底消影片首幀黑，需補各影片同名 `.png` poster。

### [x] 2. 被灯撞到：畫面震動與 SE 不同步
- **現象**：開場「被灯撞到」那拍，screen shake 與撞擊 SE 不是同時觸發，先後錯開。
- **根因（非資料）**：day1.js 撞擊節點本來就把 `shake:true, se:"impact"` 放同一 node，引擎 `playLine` 也在同一同步 tick 觸發 shake 與 playSE → **資料/引擎時序皆正確**。真因是 `playSE` 每次 `new Audio(url)` 首播需 fetch+解碼，**音檔首播延遲**，而 CSS shake 即時 → 聲音慢半拍。
- **修法（engine.js `preloadSE`，2026-06-22）**：開場 `preloadSE()` 預熱所有 SE 位元組（`new Audio().load()`），首播即時 → 撞擊聲與 shake 同拍。
- **驗證**：preview 實跑——`impact.mp3` 在進場（甚至撞擊發生前）即已 preload；無 console error。
- **備註**：目前實際存在的 SE 僅 `buzz/bump/impact` 三檔，其餘 key 仍缺檔（靜音 fallback）——屬既有素材缺口，與本條同步問題無關。
