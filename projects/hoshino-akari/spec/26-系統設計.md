## 1. 狀態變數模型

- **bond（＝現有 affection，沿用欄名）**（0…N）：累積親密／信任，決定甜瞬間密度、卸防、握手時長。是『量』不是『向』，與 stance 正交。　｜引擎成本：零破壞：直接用既有 affection 欄與所有既有 add 點。
- **seen（＝現有 awareness，沿用欄名）**（0…N）：看見『星野灯這個人而非偶像』的程度。fate_top／hidden_pov 必要；A 線靠 seen<4 與 B 線分流。　｜引擎成本：零破壞：沿用 awareness（語意即看穿）。刻意不另立 seen 欄，避免新舊體系打架。
- **regret**（0…N）：沒說出口、退太遠的累積。把結局拖向 bitter；fate_top 要 ≤1、warm_true/brave_freedom 要 ≤2。　｜引擎成本：零破壞：既有欄。
- **distance**（0…N）：情感距離，僅作 warmth=affection−distance 的內部減項。不再單獨當結局孤兒 gate。　｜引擎成本：零破壞：既有欄，既有 6 個 add:{distance} 點全保留。
- **warmth（衍生＝affection−distance）**（−N…N）：關係溫度，warm_true 主 gate（≥5）。　｜引擎成本：零破壞：不存檔、不入 initScores，evalCond 與 judge 各自即時算（現況已如此）。
- **stance（新軸，需負值）**（−N…+N（建議 −6…+6））：幫她逃(負) ↔ 逼她面對(正)。D4 後 gate cond:"stance<=-N" 分流 A/B 線；直接進命運判定。是『向』不是『量』，與 bond 正交。　｜引擎成本：破壞性中：initScores 加 stance:0；evalCond scope 加 stance。choice.add 已能加任意含負數→負值不需改 add 邏輯，只需起點與 cond 解析支援。
- **heat（新進度條 0–100）**（0…100）：有臉的第二倒數：糊照→比對→粉絲到場。越高地圖越收縮（heat>=N gate 掉場所）；真壞 gate heat>=85。　｜引擎成本：破壞性高（需正視）：initScores 加 heat:0、evalCond scope 加 heat＝小事；但『部分自動跨日上升』走離散累加裁示（見 heatDecision），不新建時間驅動子系統。
- **flag: helped_her_escape**（bool）：A 共謀逃亡線人格已立。brave_freedom 必要。　｜引擎成本：零破壞：flag 機制既有。被 judge 第3條讀。
- **flag: pushed_to_choose**（bool）：B 面對線『逼她說出想要什麼』已做。fate_top 必要，且 true_bad 必須 !pushed_to_choose。　｜引擎成本：零破壞：被 judge 第1、2條讀。
- **flag: let_her_go**（bool）：D7 放手（B＝鬆手成全／A＝成全式送走）。fate_top、brave_freedom 必要，true_bad 必須 !let_her_go。　｜引擎成本：零破壞：被 judge 第1、2、3條讀。
- **flag: gambled_on_being_seen**（bool）：D6 賭被看見（兩線皆可觸）。true_bad 必要因子。　｜引擎成本：零破壞：被 judge 第1條讀。
- **flag: sns_post_seen（沿用）**（bool）：糊照曝光線已見。hidden_pov 必要。　｜引擎成本：零破壞：既有 set 點（endings/day*）保留，被 judge 第4條讀。
- **flag: almost_confession_flag（沿用）**（bool）：差一步告白。hidden_pov 必要。　｜引擎成本：零破壞：既有，被 judge 第4條讀。

## 2. 權威 judge 偽碼（7 命運結局線性短路）

```
// meta.js judge(s, flags) —— 唯一權威線性短路；與現有 meta.js:54-61 同構，只擴充不改結構
// s.affection=bond, s.awareness=seen, s.regret, s.heat, s.stance, s.distance
judge(s, flags){
  const warmth = s.affection - s.distance;       // 沿用既有衍生式
  const bond = s.affection, seen = s.awareness;

  // 1 真壞・永久缺席（最特異，必排第一，杜絕被 warm_true 攔截）
  if (flags.gambled_on_being_seen && s.heat >= 85
      && !flags.let_her_go && !flags.pushed_to_choose)
    return "true_bad";

  // 2 命運頂・她替自己選（B 線：逼面對＋放手＋人生真改變）
  if (seen >= 4 && bond >= 5 && flags.pushed_to_choose
      && flags.let_her_go && s.regret <= 1)
    return "fate_top";

  // 3 勇敢的自由（A 線：幫逃＋成全式放手，被救非站出來）
  if (flags.helped_her_escape && flags.let_her_go
      && seen < 4 && s.regret <= 2 && bond >= 5)
    return "brave_freedom";

  // 4 隱藏・灯視點（被看見但都停在原地；沿用既有兩 flag）
  if (seen >= 4 && bond >= 5
      && flags.sns_post_seen && flags.almost_confession_flag)
    return "hidden_pov";

  // 5 暖・真（關係甜但人生零變化；舊金子降格為中段好結局）
  if (warmth >= 5 && seen >= 2 && s.regret <= 2)
    return "warm_true";

  // 6 苦・餘味（太冷太晚；主軸改 regret，廢裸 distance>=4 孤兒條件）
  if (s.regret >= 3)
    return "bitter";

  // 7 靜・常（保底，judge 最末短路）
  return "quiet_normal";
}
// 短路保證：true_bad 第1→不被暖真攔；fate_top/brave_freedom 在 hidden_pov/warm_true 前→
//   『人生真改寫』兩結局不被『關係甜但人生沒變』吞掉；fate_top(pushed_to_choose) 與
//   brave_freedom(helped_her_escape+seen<4) 靠互斥 flag 永不同時命中。
// 既同構於現況：finale() 仍可線性讀 tone；可無痛擴充。
```

## 3. 可達性矩陣（草案・待 gate-zero-A 實證）

### fate_top（命運頂・★最高）
- 條件：seen≥4 ＋ bond≥5 ＋ flag:pushed_to_choose ＋ flag:let_her_go ＋ regret≤1（短路第2，需先不命中 true_bad＝無 gambled 或 heat<85）
- 一周目觸發路徑：全程 B 偏向（stance≥+3）；D4/D6 逼真選擇都選『保她・逼她說想要什麼』(stance+, seen+, 抑 heat, set pushed_to_choose)；D6 不賭(不 set gambled)；D7 選『我幫妳走向妳自己要去的地方』成全鬆手(set let_her_go)；全程少冷淡保 regret≤1。關鍵選項 hint 暗示『這條路最苦但唯一讓她替自己活』。
- 配額：~14%（皇冠路線，刻意難但一周目精準可達）

### brave_freedom（勇敢的自由）
- 條件：flag:helped_her_escape ＋ flag:let_her_go ＋ seen<4 ＋ regret≤2 ＋ bond≥5（短路第3，需不命中前2）
- 一周目觸發路徑：全程 A 偏向(stance≤−3)：D4/D6 選『偷時間・幫她掩護・默許消失』(set helped_her_escape, bond+, heat+ 但留住 heat<85)；D6 不把賭推到 gambled+heat≥85；D7 選新開的『成全式放手：我幫妳走，但走向妳自己要去的地方』(set let_her_go)。seen 因 A 線少戳破自然<4。
- 配額：~14%

### warm_true（暖・真）
- 條件：warmth(=aff−dist)≥5 ＋ seen≥2 ＋ regret≤2，且未達上面任一更高條件（短路第5）
- 一周目觸發路徑：中性偏甜：累 bond 高、distance 低，但 D7 不選任何放手(不 set let_her_go)、也沒逼她(不 set pushed_to_choose)。修歷史傾斜：舊版僅 4.2% 可達→本版降為『不放手的高溫關係』即落點，明確可達。
- 配額：~18%（中段好結局，配額提高修傾斜）

### hidden_pov（隱藏・灯視點・皇冠成就）
- 條件：seen≥4 ＋ bond≥5 ＋ flag:sns_post_seen ＋ flag:almost_confession_flag，且未達 fate_top/brave_freedom（短路第4）
- 一周目觸發路徑：高 seen 高 bond 但都停在原地：觸發既有 sns_post_seen 與 almost_confession_flag，卻在 D7 既不 pushed_to_choose 也不 let_her_go（看穿她、差一步、最後沒動）。finale() 仍接 warm_true 正文＋hidden_pov_tail。
- 配額：~9%（隱藏，刻意稀有）

### bitter（苦・餘味）
- 條件：regret≥3，且未觸任何 free/let_go/真壞條件（短路第6）
- 一周目觸發路徑：全程偏冷：D2/D4/D6 多選退太遠／話沒說的選項累 regret≥3（既有 5 個 regret add 點＋新增）；不放手不逼她。distance 僅入 warmth，不再當孤兒 gate。
- 配額：~16%

### quiet_normal（靜・常・保底）
- 條件：fallback：以上全不命中（短路第7）
- 一周目觸發路徑：中性玩法、stance 接近 0、bond 不夠高也沒夠冷、未設任何極端 flag。『偷來的七天安靜還回去』。
- 配額：~21%（保底，自然最大宗）

### true_bad（真壞・永久缺席・★真壞）
- 條件：flag:gambled_on_being_seen ＋ heat≥85 ＋ !flag:let_her_go ＋ !flag:pushed_to_choose（短路第1）
- 一周目觸發路徑：A 線貪到底：D4/D6 每次都選『再冒一次險偷時間』把 heat 主動累加；D6 賭被看見(set gambled_on_being_seen)；D7 選 A『跟我走』挽留(不 set let_her_go)。需玩家『貪出來』——見 heatDecision：固定底數絕不單獨達 85。
- 配額：~8%（最難觸，玩家主動貪出來的懲罰，非隨機）

## 4. heat 裁定

裁定：heat 是【機制】（六軸唯一有『停不下來』潛力者，且真壞 gate heat>=85、地圖收縮 gate cond:"heat>=N" 都靠它），但其『跨日上升』走【離散累加】落地，絕不新建時間驅動子系統（引擎現無；硬上＝動 runtime 契約、BLOCKED 級風險）。

落地方式：所謂『部分自動跨日上升』改寫成『每夜開場一個固定 add:{heat:n} 的旁白節點』——資料層看似自動，實則寫死在每個 day*.js 開頭的一次 choice.add 式累加，完全走既有引擎機制、零新子系統。玩家行為疊在底數上：A 線冒險/逗留＝額外 add:{heat:+}，B 線克制＝不加或 add:{heat:-} 抑制。

每夜底數與真壞門檻（寫死、gate-zero-A 已校準）：
- 曝光線從 D3 起算（D1 後巷無人、D2 兩塊油豆腐尚未糊照）。
- 每夜固定底數 add:{heat:+10}，D3–D7 共 5 夜＝固定 +50。
- 真壞門檻 heat>=85。故單靠固定底數（封頂 50）絕不觸真壞——保證真壞是『玩家貪出來的』，不是隨機懲罰。
- 玩家須在 D4＋D6 兩個『冒險偷時間(+heat,+bond) vs 保她(抑 heat,+seen)』逼真選擇主動累加 ≥35（例：D4 冒險 +18、D6 冒險 +20＝+38；50+38=88≥85）才可能達標；謹慎玩家被動最多停在 ~50–65，安全。
- D6 賭被看見另 set flag:gambled_on_being_seen（true_bad 第二必要因子），與 heat≥85 雙鎖：兩者缺一不真壞。

可見化（不加 HUD、不動 localStorage/engine 契約）：每跨一檔用既有 ui:"sns" ＋一行旁白把『危險上升』說給玩家聽（D3 糊照貼文→D4 玻璃裡被多看→D6 同框邊角→D7 發酵），玩家不必看數字也知熱度在漲、地圖在收。與守則4協調：可見的是『威脅逼近』，克制的是『災難演法』——最壞只兌現成永久缺席，不是八卦頭條。

## 5. 分歧地圖（幫她逃 vs 逼她面對）

主控軸＝stance（幫她逃− vs 逼她面對＋），與 bond 正交（可高 bond＋任一極）。

每夜分歧累加（stance 在各夜逼真選擇分歧）：
- D1 後巷相遇（原點・揭臉反轉）：逐字保留，無 stance 分歧（定錨『我只看到一隻很急的貓』人格）。
- D2 兩塊油豆腐：起手語氣＋首個輕微 stance 傾向（縱容她偷吃− / 替她想後果＋）。
- D3 焦糖布丁：曝光線起算（開場 add:{heat:+10}）；糊照 sns 徵兆首現。
- D4 赴約的代價（中點翻牌）：第一道 PONR＋第一個 heat 真選擇。冒險偷時間=stance−、bond+、heat+；保她=stance+、seen+、抑 heat。此夜決定 D5 走 A 或 B 陣列。
- D5 她不在的那天（缺席日・神主牌）：gate cond:"stance<=-N" → A 線『兩人同行爬無人神社談私奔』全新場面 / else B 線『保留停車場痕跡解謎金子，語意升級為她在做決定』。痕跡救男主反轉兩線皆留。
- D6 並肩的雨棚（第二道 PONR）：第二個 heat 真選擇＋『賭被看見』分歧(set gambled_on_being_seen)。被拍那秒補雙向擋鏡頭錯位＋牽手改『抓了又彈開』。heat>=N 時走『無處可去』壓縮版節點。
- D7 便利店・命運結算（回原點 climax）：依 stance 與前置 flag 給不同 D7 node 與選項組——B 線『替她擋臉送她走』可 set pushed_to_choose+let_her_go；A 線除 A『跟我走』(挽留→true_bad/bitter) 另開『成全式放手：我幫妳走向妳自己要去的地方』(set helped_her_escape 已立＋let_her_go→brave_freedom 穩定可達)。

D4 後 A/B 走不同 node 陣列：用 gate cond:"stance<=-3"（A 線 then）/ else（B 線）包住 D5–D7 各自的 node 陣列；混合區（stance 近 0）走保留〈靜・常〉精神的中性節點。所有結局共享便利店甜點櫃/布丁 CG/『不要袋子』收束動作，僅末句語氣與錨點物件最終狀態不同；僅 fate_top（多一層餘味＋貓被放走＋可選車站 tail）與 true_bad（ev_vending_dark 取代甜點櫃動作）有專屬收束。

## 6. A 共謀線最小可寫量

gate-zero-B 裁定：A 共謀逃亡線 D5–D7 最小可寫量＝『與 B 線對等的三夜獨立事件』，非換語氣假分歧。最小場面清單：
- D5（A）：無人神社同行＋談私奔『全新場面』一幕（至少含一個停駐 CG 候選＋痕跡救男主的 A 線變體，互救結構保留）。
- D6（A）：雨棚『再偷一晚』變體＋heat 真選擇（賭被看見）。
