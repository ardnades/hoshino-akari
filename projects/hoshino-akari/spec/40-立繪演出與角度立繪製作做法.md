## A. 立繪演出系統 v0（引擎欄位 + 資料契約）

在 `play/data/*.js` 的 **`line` 節點**可加下列 **optional 欄位**（不寫＝完全等同舊行為，向後相容）。只動 `engine.js` + `style.css` 即可，不破壞既有資料/存檔/validator。

| 欄位 | 值 | 作用 |
|---|---|---|
| `pos` | `center`(預設) / `left` / `right` | 水平位置（CSS `translate ±11%`，保守不裁切） |
| `depth` | `normal`(預設) / `near` / `far` / `closeup` | 遠近；`near`=靠近(scale 1.10)、`far`=站遠/退半步(0.86)、`closeup`=臉部大特寫(3.6，臉部 origin，與 walk_in 終點同框) |
| `motion` | `fade_in` / `slide_in_left` / `slide_in_right` / `step_in` / `step_back` / `walk_in` / `rush_in` | 進場/靠近/退半步/走近/衝入 |
| `motionDur` | 數字(秒) | 逐句覆寫該 motion 時長（CSS 變數 `--motion-dur`；**只驅動位移/縮放，淡入是另一段固定時長、不被拖慢**） |
| `clear` | `true` | 乾淨退場（加 `.sprite-exit` 淡出 → `await` → 清空；**非 fire-and-forget**，用 `spriteGen` 競態守衛，skip/跳日安全）。注意：與 `cg:"clear"`（清 CG）不同 |

**特殊 motion**
- `walk_in`：鏡頭拉向她的**臉**慢慢放大到頭部幾乎佔滿（origin 設在畫布上方 ~20%＝臉），預設 2.6s（`motionDur` 可調）。**走近後若要維持大特寫只換表情，後續句帶 `depth:"closeup"`**（與 walk_in 終點同框，不變回原大小）。
- `rush_in`：由畫面外快速衝入＋逼近放大（撞擊）。**搭 `shake:true` + `se` 做撞畫面震動**（shake 已加強：±10px＋縱向 jitter）。

**旁白行也能顯示立繪（narration→akari 回退）**
`setSprite` 有一條回退：當 `who` 無立繪表（如 `narration`）且該 `expr` 能在 `characters.akari` 解析時，回退用 akari 立繪。→ **旁白行可用 `expr:` 顯示她的立繪**（如「轉身走進黑暗」旁白行掛 `expr:"背對"` 顯示背影，且旁白不顯示名牌標籤）。純加值，不動 akari/manager 既有行為。

**實作位置**
- `engine.js`：`SPRITE_POS/SPRITE_DEPTH/SPRITE_MOTION` 對照表、`spriteGen`、`setSprite(who,expr,mask,opts)`（讀 opts.pos/depth/motion/motionDur→套 class）、`clearSpriteAnimated()`、narration→akari 回退。
- `style.css`：`.sprite-pos-left/right`（`translate`）、`.sprite-depth-near/far/closeup`（`scale`，closeup 另設臉部 `transform-origin`）、`.sprite-motion-*`（keyframe，時長吃 `var(--motion-dur, 預設)`）、`.sprite-exit`。
  - 關鍵：**pos 用 `translate`、depth 用 `scale`（獨立屬性，互不覆蓋、可疊加）；motion 用 `transform` keyframe**，收尾於 identity → `prefers-reduced-motion` 下（通用規則壓 duration）落在靜止態，無障礙安全。
- 純資料就能做（零引擎）：`pause`、`speed:slow/instant`、expr 順序、HOLD（省略 expr→立繪續留）、`expr:""`（清）、`screen:"black"`。

**測試頁**：`play/_stage_test.html`（stub 資料驅動真引擎，逐項試 pos/depth/motion/clear/walk_in/rush_in；非正式檔）。

## B. 角度立繪生產做法（Anima/ComfyUI）＋ 關鍵教訓

**模型/管線**：ComfyUI `http://127.0.0.1:8000`、底模 `anima_baseV10.safetensors`（Cosmos-Predict2，**無 ControlNet/IP-Adapter**）。可用 LoRA：角色鎖 `akari_charlock_v2-000020`、`akari_hoshino_anima_v1-*`；風格鏈 `ayori_v1_step3400`、`gpt-image-2_anima-base1_v1`、`AnimaNEWNSS8`、`dogma_animaV1.6`、`V67_Watercolor_Anime_*`。出圖參數：er_sde/beta、cfg4、~32 步、832×1216；觸發詞 `@ay0r1, @gpt-image-2`。

**關鍵教訓（角度立繪）**
1. **3/4 背是甜蜜點**：長髮自然落到側邊→不穿幫、平視可控、最像她。**純正背面＋長髮會穿幫**（長髮垂在背中央會跟白圍巾/連帽衣層次打架，看起來像穿過衣服）。GPT 也背書：3/4 為主力角度，純正背面就用「收髮版」當分工。
2. **角色 LoRA 用正面長髮訓練 → 會跟角度/帽下髮打架**。對「臉幾乎不露」的角度（背/3-4），**可不用角色 LoRA**（靠 outfit＋風格鏈＋canon prompt 就夠像，且不穿幫、角度自由）；要更濃 canon 感再加回 LoRA（~0.5–0.75）。正面臉/特寫才一定要 LoRA。
3. **平視 vs 俯視**：Anima 沒裝 ControlNet/OpenPose，所以角度/平視**靠 prompt（"eye level, NOT from above"）＋ seed** 控制，多 seed 挑。
4. **頭髮穿幫的根源**：長髮垂背 ＋ 多層服裝（帽/圍巾/連帽衣）。解法：撥髮到一側、或收髮、或局部 inpaint 修（見 C）。
5. **走路全身姿勢**易跑成俯視＋圍巾漂移，要的話多 seed 並嚴格 prompt（"mid-stride, hands in pockets, eye level"）。

**生產腳本樣式**（繞過 generation_service 的角色必填，直接用 comfy_client）：
```
config.build_app_config() → comfy_client.health_check / get_loras
comfy_client.load_workflow_template(art_workflows/base_txt2img_api.json)
comfy_client.patch_workflow(tpl, positive, negative, checkpoint, w, h, seed, steps, cfg,
    sampler="er_sde", scheduler="beta", loras=[...], available_loras=get_loras())
comfy_client.queue_prompt(url, patched.workflow, client_id)
comfy_client.wait_for_result(url, pid, http_get=comfy_client._default_http_get, sleep=time.sleep, ...)  # ★ http_get/sleep 是必填 kwarg
image_importer.parse_comfy_history_for_images / fetch_comfy_image(url, ref, http_get=...)  # ★ http_get 必填
```
參考腳本：`art_tool/_backview_*.py`（角度立繪）、`art_tool/_bg_trial.py`（背景，無人物用 base+場景 prompt 即可、不需角色 LoRA）。

## C. 局部修圖做法：手塗粗刪 → ComfyUI masked inpaint（★ 已驗證好用）

當一張圖大致對、只有局部要改（如要刪掉穿幫的那撮髮）：

1. **手塗粗刪**：在原圖上把不要的部分用接近底色粗略蓋掉（不用精細），存成另一檔（如 `*_b.png`）。
2. **建 mask（精準）**：用「塗過版 vs 原版」做像素差 → 差異區＝塗抹範圍 → 膨脹＋羽化。避免手畫 mask。
3. **masked img2img inpaint**：用 `art_tool/art_workflows/face_inpaint_api.json`（VAEEncode + **SetLatentNoiseMask**，遮罩外 latent 凍結；**不要用 VAEEncodeForInpaint**＝會抹掉原內容）。patch：節點30 來源圖、31 mask、6/7 prompt、3 seed/denoise。denoise ~0.6，prompt 描述「該區應該長什麼」（如「乾淨淺藍連帽衣、無頭髮」）。
4. 參考腳本：`art_tool/_inpaint_back.py`（含像素差 mask + 上傳 `/upload/image` + patch + queue/fetch）。

## D. 去背做法

**首選：`art_tool/character_pipeline/batch_cutout.py`**（演算法白底去背：邊界連通白底移除 + 內部負空間白洞填補 + 髮 alpha review）。
- **白圍巾優勢**：白圍巾被主體包住、不連到邊框 → 當前景保留（比 RMBG 對「白對白」更穩）。
- 用法：`python character_pipeline/batch_cutout.py --files "<圖>" --output "<dir>" --batch-name NAME --overwrite`
- 輸出：`<dir>/<batch>/alpha/*_alpha.png`（透明 PNG）＋ `qc/*_qc_dark|checker|midgray.png`（去背檢查預覽）＋ `review/`（髮緣紅/黃/青標記，只標不改）。
- 需 `numpy` + `Pillow` + `scipy`。規格務必 **832×1216 RGBA**。
- 商用替代（若日後需要）：BiRefNet（Apache/MIT）；RMBG-1.4 為非商用。

## E. drop-in 接入做法（零/低契約）

**立繪**
1. 去背 PNG（832×1216 透明）複製進 `play/assets/characters/`（命名沿用 `akari_cb_<expr或角度>.png`）。
2. `assets.js` → `characters.akari["<中文 expr>"] = "assets/characters/xxx.png"`。
3. 劇情 `line` 用 `expr:"<中文 expr>"`（需口罩再加 `mask:"口罩"/"半罩"`）。**旁白行**也能顯示（靠 A 的 narration→akari 回退）。
4. 角度立繪是整圖去背、**不疊口罩 overlay**（背影/側面不需要）。

**背景**（你 session 已加逐場景背景）：`assets.background[mood]`（mood 後備）或具名 key（`bg_<place>_<tone>`）；`scene.bg` / `line.bg` 指定地點背景（覆蓋 mood 後備），缺則 CSS 漸層。

**道具/CG**：`assets.cg[key]`（`oden/pudding/cocoa/lipbalm/receipt/note/cat_meet/akari_studio` 等既有 key＝換圖零契約）。

**收尾必做**
- `node projects/hoshino-akari/tools/validate_story.js` → 維持 **0 error / 0 warning**（新演出欄位/expr 不被 validator 擋）。
- 改檔後 **bump `index.html` 的 `?v=` 版本號**（否則瀏覽器快取舊檔；或硬刷 Ctrl+F5）。PNG 等無版本號的素材，硬刷即更新。

## F. 範例：Day1 結尾離場（已實裝，可當樣板）

## H. 正面立繪藍挑染補齊（blue-highlight overlay；不重造）

canon = 黑髮 ＋ **冷藍挑染**（lock-v3），但正面 cb_ 立繪（neutral/smile/pouty/shy/accept/troubled）生成時藍太弱、與角度圖（壓帽 s89104 等）不一致。腳本：`art_tool/_hairblue_trial.py`。

- **★ 關鍵教訓**：**灯的黑髮本身帶冷藍色調**（色相落在藍區、S 不低），所以「用色相排藍」會把頭髮自己一起排掉，且**分不出藍黑髮 vs 淡藍連帽衣**（兩者同色相）。→ **改用亮度分**：暗（V<~118）＝髮、淡藍連帽衣較亮被排除；再用**左右側帶**（排中央白圍巾）。早期沒這兩道 → 藍溢到圍巾/連帽衣暗部。
- **`lock` 模式（單束實心藍，最像角度圖）**：只染**下半垂髮**（y 0.46–0.85）、整束實心冷藍（H152 / S235 ＋ LIFT 提亮，不靠受光），羽化。
- **輸出可疊加層** `<name>_bluelock_LAYER.png`：RGB＝滿染藍、A＝羽化權重、其餘全透明，同 832×1216 對齊 → **直接以 normal 疊到任一正面 base**（6 張共用 master body，**一張 overlay 通用**）；使用者再加工。
- `subtle/med/strong` 模式＝散布受光髮絲的較淡挑染（lit-weighted、雙側），給不要單束、只要整體偏藍時用。
- 髮束 x/y 位置（`LOCKS`/`LOCK_Y_*`）是依正面 cowboy master body 框定；換別的構圖要重調並看 `*_bluemask_overlay.png` 確認落位。
